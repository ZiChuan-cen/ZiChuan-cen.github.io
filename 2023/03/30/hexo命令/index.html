HEXO 常用命令

经典的hexo三联

下面是详解：

hexo clean

清除已经生成的文件（清除旧有的文件，防止产生冲突）

hexo g

生成静态文件

hexo d

部署到服务器 比如说：当你选择把博客放在GitHub时，运行hexo d 指令时就会执行上传指令

接下来是hexo s

当执行完hexo g 命令后想要先预览一下，可以执行hexo s 命令运行本地服务器，接着打开浏览器，在地址栏输入 http://localhost:4000/ 即可预览。
在预览的同时，你也可以继续修改自己不满意的地方，此时进行的修改会在浏览器上同步呈现。
当你想结束预览时，只需在git命令行按 Ctrl+C 即可。

文件操作

hexo n “文章名称” / hexo new “文章名称”

新建文章，文章名称自定义。

hexo new page “名称”

新建分页

hexo new draft “名称” / hexo n draft “名称”

新建草稿

hexo publish “名称” / hexo p “名称”

草稿生成文章

hexo publish page “名称” / hexo p page “名称”

草稿生成分页

注意：以上hexo 命令均在git 命令行中输入并执行！
